Polyfills

Exifr is built using the latest standards and web platform features. We use Babel to transpile the new syntax down to ES5; Rollup to bundle all ESM files into single UMD bundle usable in older browsers. But it's up to you to include polyfills in your app.

In this example we use:

  • polyfill.io with presets es2015, es2016, es2017 and polyfills Object.fromEntries, fetch
  • TextDecoder
  • regeneratorRuntime (needed by babel)

Needs to be polyfilled:

If you decide to use core-js or another polyfill, here's a list of some of the features exifr uses.

    Object.assign, Object.keys, Object.values, Object.fromEntries

    Array.from, Array.prototype.includes

    fetch

    TextDecoder

    Map, Set

    Uint8Array, DataView, ArrayBuffer
